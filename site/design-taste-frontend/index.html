It reads the room before it writes a line.

The bans are not a fixed style. Before any code, the skill states a one-line design read: page kind, audience, vibe, and the system or aesthetic it is leaning toward. The tells are checked against that read, not against a default look.

the design read, declared first
# one line, before generating anything
Reading this as: B2B SaaS landing for
technical buyers, with a Linear-style
minimalist language, leaning toward
Tailwind + Geist + restrained motion.
# three dials set from the read
VARIANCE 6 · MOTION 4 · DENSITY 3
# then the preflight runs, every box
zero em-dashes · one accent · hero fits

Run it as your frontend preflight.

It is for landing pages, portfolios, and redesigns, not dashboards or data tables. Point it at a brief, it declares the design read, picks a real system or an honest aesthetic, and runs the full preflight before the page ships.

  • Infers the brief and declares a one-line design read first.
  • Bans the AI tells: em-dashes, equal cards, fake screenshots, AI purple.
  • Closes with a mechanical preflight check, every box ticked.
