An agent under pressure will say "done" early.

Long pipelines fail at the seams. The idea was never sharpened, the spec lived in chat scrollback, the tests came after the code, and the final "everything passes" was a sentence, not a result. Each seam leaks a little slop, and the sum ships.

Crucible makes every seam a file and every claim a check. A stage cannot start without its predecessor's artifact, and the run cannot end until each gate records a passing exit code plus the output that proves it. The model does not get a vote; the ledger does.

Seven stages, seven artifacts.

Each stage consumes the file before it and writes its own, so a run can pause, resume, or restart from any boundary. The specialists it conducts are the other skills in this marketplace.

Build

Per-task subagents drive the tests green via subagent-driven-development, in parallel where tasks allow it, in worktrees where isolation matters.

50-build-log.md
50
Assay

Five gates: tests, lint, typecheck, a spec review, and a fresh-context slop critique. Failures go through systematic-debugging and re-record until the ledger says pass.

Deliver

Only after ASSAY PASS can the run flip to done. The summary records what shipped, the key decisions, and how each gate was proven.
