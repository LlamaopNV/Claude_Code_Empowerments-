The diagram only exists if you draw it twice.

Mermaid keeps your diagrams as text, versioned next to the doc. But a reviewer opening the PDF sees a fenced code block, not a picture. This skill renders the blocks and pages the whole document for you.

Interactive

Pick a diagram. Watch it become a page.

The left pane is the real mermaid source. On play, mmdc renders the fenced block to a true SVG with mermaid.js, then md-to-pdf lays the document out on an A4 sheet. The code and the diagram genuinely match.

what runs under the hood
mmdc -i design.md -o design-rendered.md # fenced mermaid to SVG
npx md-to-pdf design-rendered.md # A4, print background on

Point it at a Markdown file.

Give it the absolute path to a doc with mermaid blocks. It renders each block to an SVG, writes a rendered copy of the Markdown, then converts that to a paged PDF. Your original file is left untouched.

  • Every mermaid block becomes a co-located SVG image.
  • The whole doc is paged to A4 with margins and print backgrounds.
  • The source Markdown is never modified.
