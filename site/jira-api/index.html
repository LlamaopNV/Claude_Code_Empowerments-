The API is the easy part. The plumbing isn't.

Basic-auth headers, the right base URL, the moved search endpoint, transition IDs you have to look up, and comments that must arrive as ADF. jira-api absorbs all of it so a skill just asks for the issue.

Wire it once, then just ask.

It is a reference plus scripts skill, so it loads when another skill needs Jira. Set five env values in your project, and the scripts handle auth, URLs, and error checking on every call.

  • Auth from JIRA_BASE_URL, JIRA_API_TOKEN, JIRA_EMAIL.
  • Search, CRUD, transitions, comments, sprints, and epics.
  • Plain text wraps into ADF for you on every comment.
set up and call
# set per project in .claude/settings.json env
JIRA_BASE_URL=https://your-org.atlassian.net
JIRA_API_TOKEN=your-token
JIRA_PROJECT=ABC JIRA_BOARD=42
# the scripts live in the skill dir
${CLAUDE_SKILL_DIR}/scripts/jira-search.sh
# or just ask, the skill loads itself
"what is in the active sprint?"
"move ABC-42 to In Progress and comment"
