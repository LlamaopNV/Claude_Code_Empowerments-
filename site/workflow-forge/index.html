Every repo relearns the same rules.

A fresh checkout has no CLAUDE.md, no test gate, no convention to point at. So the model guesses your commands and your house style. workflow-forge detects the stack instead of pasting a template, and wires the disciplines in once.

Pick a stack. Forge the workflow.

The detector reports the host OS and toolchain. workflow-forge fills the CLAUDE.md from the real commands it finds, then clicks each guardrail into place. Switch the stack or OS and the output adapts.

Bootstrap any repo.

It does not ship a fixed CLAUDE.md. It generates one into the repo it runs in, adapting to the detected stack and host OS, and never overwrites an existing CLAUDE.md without your go-ahead.

  • Detects JS/TS, .NET, Rust, Go, and Python, plus the host OS.
  • Wires in the TDD guard, pre-commit gate, and symmetric audit.
  • Seeds TODO.md and WORKLOG.md and keeps them honest.
